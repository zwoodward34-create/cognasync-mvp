# Examples of Wellness Features That Trigger Medical Device Status

Many features that **seem** like simple wellness tools can cross into medical device territory based on their **functionality, claims, or intended use**. Below are concrete examples organized by feature type.

***

## 1. Mental Health Features

### ✅ Wellness (Non-SaMD)

| Feature | Example | Why It's Safe |
| :-- | :-- | :-- |
| **Mood tracking** | User enters: "Mood: 6/10" | Simple data collection [^11] |
| **Mood averages** | "Average mood: 6.2 (range: 3-9)" | Simple arithmetic (easily verifiable) [^11] |
| **Trend charts** | Line chart showing mood over time | Shows what happened, no interpretation [^11] |
| **Journaling prompts** | "Choose from 3 journaling prompts" | User selects content, not AI [^11] |
| **Stress management** | "Guided meditation for relaxation" | General wellness claim [^3] |

### ❌ Triggers Medical Device Status

| Feature | Why It Crosses the Line | Regulatory Risk |
| :-- | :-- | :-- |
| **"AI detects depression patterns"** | AI/ML automatically triggers SaMD [^12] | Diagnostic claim [^11] |
| **"Risk score: 73%"** | Algorithmic risk scoring = SaMD [^11] | Predictive/clinical claim [^11] |
| **"Symptoms worsening" warning** | Clinical interpretation = SaMD [^11] | Monitoring claim [^11] |
| **"We recommend CBT exercise X"** | Treatment recommendation = SaMD [^11] | Treatment claim |
| **"Contact your therapist now"** | Clinical triage = SaMD [^11] | Medical management [^1] |
| **"Predicts mood will worsen"** | Predictive model = SaMD [^12] | Prediction claim |
| **"Flags high-risk users"** | Clinical decision support = SaMD [^11] | Risk assessment [^11] |

| Feature Type | Wellness (Safe) | Medical Device (Risky) |
| :-- | :-- | :-- |
| **AI/ML** | ❌ None (AI = automatic SaMD) | AI/ML in any analysis [^12] |
| **Risk scoring** | ❌ None (risk = SaMD) | "Risk score: 73%" [^11] |
| **Clinical triage** | ❌ None (triage = SaMD) | "Contact therapist now" [^11] |

| **Treatment** | ❌ None (treatment = SaMD) | "Treats anxiety" |

## Key Takeaways

| Rule | Implementation |
| :-- | :-- |
| **AI/ML = automatic SaMD** | Never use machine learning [^12] |
| **Risk scores = SaMD** | Use raw counts only (e.g., "3 days with mood < 4") [^11] |
| **Clinical triage = SaMD** | Provide crisis resources only, no algorithmic routing [^11] |
| **Clinical interpretation = SaMD** | Show "what happened", not "what it means" [^11] |
| **Most predictions = SaMD** | Use descriptive trends only [^12] |
| **Diagnosis/treatment = SaMD** | Never make medical claims [^11] |
| **Disease population = SaMD** | Wellness apps for general population only [^2] |
| **Invasive technology = higher risk** | Non-invasive sensors only [^1] |

**Bottom line:** Features that trigger SaMD include AI/ML, risk scoring, clinical triage, clinical interpretation, predictions, diagnosis, treatment, and monitoring of medical conditions. Stay in wellness territory by using simple arithmetic, descriptive trends only, and wellness-appropriate claims.[^12][^11]
