## Work Stream 1 — Non-Respondent Flagging System

### Background
Prior to this session, patient silence was treated as absent data — gaps were noted by calendar day but not surfaced as a clinical signal in their own right. There was no concept of a "prompted non-response" (i.e., a prompt was sent but ignored), no 5-day streak flag, and no threshold that warned the provider when there wasn't enough engagement to trust the summary.

### What was built

#### `database.py` — `compute_engagement_stats()`
Three new computed fields added to the return dict:

**1. Per-channel unanswered dates**
`sms_by_flow[channel]['unanswered_dates']` — a sorted list of ISO date strings where a prompt of that channel type was sent (`created_at`) but `used_at` was NULL. Covers all four channels: `medication`, `short`, `full`, `voice`.

**2. Prompt-streak tracking**
- `max_prompt_gap` (int) — longest consecutive run of calendar days where at least one prompt was sent and none were answered
- `prompt_gap_segments` (list) — `[{start, end, days}]` for every streak ≥ 5 days
- `extended_no_response` (bool) — `True` when `max_prompt_gap >= 5`

Distinct from the existing `max_consecutive_gap` (which counts calendar-day silence regardless of whether a prompt was sent).

**3. Insufficient-data flag**
- `overall_sms_rate` (float) — response rate across all channels combined
- `insufficient_data` (bool) — `True` when `overall_sms_rate < 0.40` AND at least 3 prompts were sent (the ≥3 guard prevents false positives on newly enrolled patients)

#### `claude_api.py` — Mode C prompt injection
Both `generate_appointment_summary(audience='provider')` and `generate_psychiatric_summary()` were updated. New variables unpacked from `engagement_data`:
```python
extended_no_response, prompt_gap_segs, max_prompt_gap, overall_sms_rate, insufficient_data
```

Three new system prompt instructions injected when the signal is present:

**Per-channel unanswered dates** — rendered inline in the Engagement subsection:
```
• medication adherence: 8 of 14 (57%) — unanswered on: 2025-04-03, 2025-04-07, ...
• voice recording: 0 of 4 (0%) — unanswered on: 2025-04-01, 2025-04-08, ...
```

**Extended no-response streak** — 🔴 flag in Flags section:
```
🔴 Extended non-response — [N] consecutive days without responding to any SMS prompt ([start] – [end]). Clinical check-in recommended.
```

**Insufficient data** — 🔴 flag pinned to TOP of Flags section, plus one cautionary sentence in Trajectory. The rest of the summary is NOT suppressed — what data exists is surfaced, clearly flagged as limited.
```
🔴 ⚠ Insufficient data — overall response rate is [pct]% across all channels. 
Pattern observations in this summary are based on a minority of expected data 
points and should be interpreted with caution.
```

#### `CLAUDE.md` — Section 26 added
Full spec documenting:
- The four provider-facing signals (unanswered dates, 5-day streak, selective channel, insufficient data)
- Thresholds and flag levels for each
- Mode C and Mode D output formats
- Language rules (e.g., "unanswered on" not "missed on"; never speculate about cause)
- What must never appear in patient-facing output (Mode A / Mode B)
- Integration points

### Design decisions made
- **40% threshold for insufficient data** — chosen by Tim; only activates after ≥3 prompts sent
- **5-day streak threshold** — chosen by Tim; based on consecutive *prompted* days, not calendar days
- **Summary is not suppressed at <40%** — flagged but not hidden; provider still sees what data exists
- **Patient-facing output is never affected** — all four signals are provider-only

## Work Stream 2 — Patient Hub UI Polish

### File changed
`templates/provider/patient_hub.html`

### Changes made

**1. Removed Medication Adherence card from Detailed Records sidebar**  
The right-column "Detailed Records" section previously included a collapsible Medication Adherence card (with a mini calendar and edit/delete buttons per medication). This was removed entirely. Medication adherence data is still visible in the main Clinical Data panel under the "Meds" tab.

**2. Date numbers inside adherence grid cells**  
The 4-week medication adherence calendar grid previously rendered blank colored squares. Each cell now displays the day-of-month number (e.g., "3", "14", "28") so the provider can read specific dates at a glance without hovering for the tooltip.

JS change in `buildAdherenceGrid()`:
```js
// Before
weeksHtml += '<div class="ch-adh-cell ' + cls + '" title="' + day + '"></div>';

// After
var dayNum = parseInt(day.slice(8), 10);
weeksHtml += '<div class="ch-adh-cell ' + cls + '" title="' + day + '">' + dayNum + '</div>';
```

**3. Legend labels updated**  
Colors were already correct. Labels updated to match the semantic intent:
- "Missed" → **"Not taken"** (medication was logged, patient said no)
- "No data" → **"No response"** (no SMS response; medication status unknown)
